Project Initializer

Enterprise Documentation Portal

Production-grade guide for generating, validating, and operating GitOps-ready platform projects with Flux/Argo, Terraform, ECK, and RKE2-oriented bootstrap workflows.

1. Overview

Project Initializer creates repeatable infrastructure/project skeletons from a single form or API call, including platform manifests, GitOps controllers, automation scripts, and operational runbooks.

2. Quick Start

  1. Open the UI front page at http://localhost:8787.
  2. Create project with target platform + GitOps + IaC options.
  3. Run generated deployment scripts in order (bootstrap, terraform, post-deploy).
  4. Verify minimum cluster health with scripts/cluster-healthcheck.sh.
./start-ui.sh
# then open http://localhost:8787

3. Architecture

UI + API

  • Frontend: single-page UI in ui-draft/frontend/index.html.
  • Backend: FastAPI service in ui-draft/backend/backend_api.py.
  • Generated output: project folders with manifests/scripts/docs for deployment lifecycle.

Deployment Model

  • Root GitOps kustomization reconciles infra before apps.
  • Agents are isolated in dedicated Flux kustomization for independent status/reconcile.
  • Kibana ingress and ECK network policies are generated automatically.

4. Operations and Reliability

  • Use explicit Flux timeouts for heavy first syncs (ECK stack bootstrap).
  • Use healthcheck script to validate nodes, pods, ECK resources, ingress, credentials, Flux state.
  • For tier placement, provide node_selector per hot/cold/frozen nodeSet in sizing context.
{
  "data_nodes": { "node_selector": { "node-role.kubernetes.io/hot": "true" } },
  "cold_nodes": { "node_selector": { "node-role.kubernetes.io/cold": "true" } },
  "frozen_nodes": { "node_selector": { "node-role.kubernetes.io/frozen": "true" } }
}
