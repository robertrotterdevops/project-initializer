<objective>
Fix the Flux manifest generator to produce structurally correct Kustomization CRs that match the es-06 reference deployment. The primary bug is that `_calculate_complexity()` in `flux_deployment.py` varies the reconciliation interval (5m vs 1m) and gotk-sync timeout (2m vs 5m) based on keywords in the project description. This causes incorrect Flux CR values for "advanced/enterprise/multi-cluster" descriptions, breaking reconciliation.

Purpose: Eliminate complexity-dependent timeout/interval variability that causes Flux kustomizations to hang or behave incorrectly on real clusters.
Output: Fixed `addons/flux_deployment.py` with hardcoded es-06-reference-matching values.
</objective>
